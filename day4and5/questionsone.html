1 Q.)Differences between var let and const variables AND HOISTING


        VAR:
        i.)var variables are global scope variables.
        ii.)var variables can be hoisted.
        ii.)var variables can be redeclared.
        iii.)var variables can be updated.
        iv.)var variables once declared,can be initialised later also.

        

        LET:
        i.)let variables are block  scope variables.
        ii.)let variables can be hoisted,but they are hoisted to the temporal dead zone.
        ii.)let variables cannot be redeclared.
        iii.)let variables can be updated.
        iv.)let variables once declared,can be initialised later also.

        

        CONST:
        i.)const variables are block  scope variables.
        ii.)const variables can be hoisted,but they are hoisted to the temporal dead zone.
        ii.)const variables cannot be redeclared.
        iii.)const variables cannot be updated.
        iv.)const variables should be declared and initialised at the same time.

        

        

        HOISTING:Hoisting is a default behaviour of javascript to push the decalrations to the top.
        

        TEMPORAL DEAD ZONE: When let and const variables are declared ,they are said to be  in temporal dead zone until unless they get initialised with a value.

    
         2Q.)In Execution or global execution context we have two phases i.)Creation phase and ii.)Execution phase
     
        I.)Creation phase: In creation phase the compiler runs through the entire code , so in the first iteration it pick ups all the function declarations and store them in the memory with their reference ,
         and in the second iteration it picks up all the variables and assign them undefined.
     
         II.)Execution phase: In execution phase it assigns the variable with a value and starts executing the functions.
     
         

         

       
         

         

     
          Call Stack, Call back queue,Microtask queue,Event Loop,WebApi,
          Consider we have a Settimeout function which is a WebApi.Since Settimeout is asynchronous function , 
          in javascript the asynchronous functions are executed in a different way,they will not interfere with the synchronous tasks.
          The settimeout function is passed to the call back queue,Call back queue by the name itself will have the callback functions which are yet to be executed,
          Microtask queue also does the same thing but Microtask queue takes Promises,Microtask queue has more priority than Call back queue.
          The Event loop acts like a mediator between (callback queue ,microtask queue) and (Call stack).Event loop tasks is to send the call back functions to call stack for execution once call stack gets empty.
          Call Stack keeps the tracking of functions.And function is removed from the call stack once the function gets returned or when the functions reaches its end of the scope or end point.
